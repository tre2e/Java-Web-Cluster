# 🚀 项目介绍：Java Web Cluster - 智能广告投放网站群

## 🎯 项目概述

本项目旨在构建一个**基于智能广告投放的 Java Web 网站集群**，通过连接一个核心**广告投放网站**与三个面向不同用户群体的垂直应用网站（**购物**、**视频**、**新闻**），形成一个完整的生态系统。它不仅是一个实验性作业，更是一个涵盖现代 Web 开发核心技术的综合性平台，目标是实现**无登录状态下的跨站兴趣追踪**与**个性化广告精准投放**。

## 🌐 网站集群构成与功能详情

本项目由一个核心管理网站和三个用户端网站组成，共同实现广告的投放、展示和效果追踪。

### 1. 📢 广告投放网站（面向广告投放方 / 管理员）

- **核心功能**：广告管理、效果追踪、内容分发、智能投放。
- **管理员操作**：
  - 投放、编辑、管理**文字**、**图片**、**视频**三种形式的广告；
  - 为每个广告设置**兴趣标签**（如 `digital`、`fashion`、`sports`）；
  - 实时查看各广告的**点击量**、**曝光次数**等核心指标。
- **智能投放**：根据用户兴趣模型，动态向购物、视频、新闻网站推送匹配的广告内容。

### 2. 🛍️ 购物网站（面向顾客群体）

- **核心功能**：商品浏览、内容展示、广告嵌入。
- **用户体验**：游客可自由浏览商品信息，页面自动加载个性化广告。
- **广告集成**：在首页及商品列表页嵌入来自广告平台的动态广告位。

### 3. 📺 视频网站（面向视频观众）

- **核心功能**：视频聚合、播放、主页广告展示。
- **用户体验**：用户可浏览视频缩略图并点击观看。
- **广告集成**：在主页面关键区域（如推荐位上方）展示高相关性广告。

### 4. 📰 新闻网站（面向新闻读者）

- **核心功能**：新闻分类阅读、内容推送、侧边栏广告。
- **用户体验**：按类别（科技、体育、财经等）筛选新闻。
- **广告集成**：在新闻详情页两侧边栏动态插入兴趣匹配广告。

## 🔍 核心亮点：无登录跨站兴趣追踪机制

> **如何在用户不登录、四站完全独立的前提下，实现“看过数码新闻 → 视频站推手机广告”？**

本项目创新性地采用 **浏览器指纹（Browser Fingerprinting） + 中央行为上报** 的方案，实现**匿名化跨站用户识别与兴趣建模**：

1. **指纹生成**：  
   用户首次访问任一网站时，前端通过轻量级 JS 脚本采集设备特征（User-Agent、屏幕分辨率、时区、语言等），生成唯一且稳定的哈希值（如 `fp_a1b2c3d4`）。

2. **行为上报**：  
   各网站在用户浏览内容时（如阅读“数码新闻”），自动将 **指纹 ID + 内容类别** 上报至广告管理平台。

3. **兴趣建模**：  
   广告平台以指纹 ID 为键，聚合用户在多个站点的行为，构建实时兴趣画像（如“近期高频关注数码类内容”）。

4. **精准投放**：  
   当用户访问其他站点时，前端携带相同指纹请求广告，平台据此返回匹配兴趣的广告内容。

✅ **优势**：
- 无需用户登录，无 Cookie 依赖，适应现代浏览器隐私策略；
- 四个网站彼此无任何跳转或通信，完全满足“独立部署”要求；
- 兴趣基于**真实浏览行为**，而非仅依赖广告点击，推荐更精准。

## 💡 项目价值与总结

1. **数据驱动**：实现从行为采集、兴趣建模到广告推荐的闭环；
2. **隐私合规**：采用匿名指纹技术，在无登录前提下保障用户隐私；
3. **系统解耦**：四站完全独立，仅通过广告平台间接协同，架构清晰；
4. **扩展性强**：支持多类型广告、多维度兴趣标签，易于后续迭代。
